## 💡 Key Insight

Demand is highly dependent on:
- Previous sales (lag features)
- Weekly seasonality
- Promotions

This project demonstrates how simple feature engineering can significantly improve forecasting accuracy.

# Retail Demand Forecasting Engine 📈

##  Overview
Built a machine learning system to forecast retail product demand using historical sales data, pricing signals, and time-based features.

## 🎯 Problem
Retailers struggle with:
- Overstocking
- Stockouts
- Poor demand planning

##  Solution
Developed a Random Forest-based forecasting engine that:
- Captures temporal patterns
- Uses lag and rolling features
- Predicts future demand accurately

## ⚙️ Tech Stack
- Python
- Pandas
- Scikit-learn
- Streamlit

## 📊 Features
- Time-series feature engineering
- Lag-based forecasting
- Multi-store support
- Interactive UI for predictions

## 📈 Results
- Reduced forecasting error by ~20-30%
- Improved demand visibility
